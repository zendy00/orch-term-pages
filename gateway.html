소개

게이트웨이 · HTTP · OpenAI 호환

AI 게이트웨이는 외부 프로그램·스크립트가 로컬 HTTP로 Orch term 안의 AI에게 프롬프트를 보내 결과를 동기로 돌려받는 기능입니다. 단발 호출도, session으로 멀티턴 대화도 됩니다. OpenAI chat.completion 호환 응답이라 기존 OpenAI 클라이언트도 base URL만 바꿔 붙일 수 있습니다.

  • 프로토콜: HTTP, 기본 포트 9610. 보안: 127.0.0.1 + Bearer 토큰.
  • 기본 활성화: 신규 설치 시 on(설정에서 끄거나 다시 켤 수 있음).
  • 엔드포인트: POST /run(또는 OpenAI 별칭 POST /v1/chat/completions) — 폴링 없음, 동기.
AI가 다른 AI를 워커로 띄워 협업·조율시키는 건 게이트웨이가 아니라 오케스트레이션(지휘자) 기능입니다 → 오케스트레이션 소개 페이지.

워커 실행 폴더 · 최초 1회 승인

게이트웨이 요청을 받으면 안에서 AI 워커(claude·codex·agy)가 고정된 작업 폴더에서 실행됩니다.

실행 폴더(고정): ~/.orchterm/gateway — 설정에서 변경할 수 있습니다.

최초 1회 승인 필요 — 이 폴더에서 각 에이전트가 처음 실행될 때 "이 폴더를 신뢰하시겠습니까?"(folder trust) 확인이 워커 탭에 한 번 뜹니다. 한 번 승인하면 폴더 경로가 고정이라 이후에는 다시 묻지 않습니다. (claude·codex는 정확한 경로 단위로, agy는 상위 경로 prefix로 신뢰가 저장됩니다.) 첫 게이트웨이 호출이 응답 없이 멈춘 것 같으면, 워커 탭에서 신뢰 승인을 한 번 처리해 주세요.

폴더는 설정 → AI 게이트웨이 → 워커 폴더에서 원하는 경로로 바꿀 수 있습니다(다음 호출부터 적용). 바꾸면 새 폴더를 다시 1회 승인하면 됩니다.

안전 모드 기본 — 위험 권한 우회 플래그를 쓰지 않습니다(그래서 위 1회 승인이 필요). 무인 자동화로 승인 없이 바로 돌리려면 설정 → AI 게이트웨이 → "워커 위험 권한 우회"를 켜세요(빠르지만 비권장). 안전 모드의 claude는 이 폴더에 자동 생성된 .claude/settings.local.json으로 파일/명령 프롬프트가 폴더 한정 억제됩니다.

curl / PowerShell 예제

포트·토큰은 앱의 설정 → AI 게이트웨이에서 확인합니다(기본 포트 9610). 토큰은 항상 Authorization: Bearer <토큰> 헤더로 전송됩니다.

curl

curl -s -X POST http://127.0.0.1:9610/run \
  -H "Authorization: Bearer <토큰>" \
  -H "Content-Type: application/json" \
  -d '{"agent":"claude","prompt":"안녕하세요. 자기소개 해주세요."}'

PowerShell

$h = @{ Authorization = 'Bearer <토큰>' }
$body = '{"agent":"claude","prompt":"안녕하세요. 자기소개 해주세요."}'
$r = Invoke-RestMethod -Method Post -Uri 'http://127.0.0.1:9610/run' -Headers $h -ContentType 'application/json' -Body $body
$r.choices[0].message.content
응답은 동기입니다 — 에이전트가 답할 때까지(최대 약 6분) 연결이 열려 있다가 OpenAI chat.completion 형태로 돌아옵니다.

session · 대화 이어가기(멀티턴)

session 필드가 핵심입니다 — 워커를 1회성으로 쓸지, 대화를 이어갈지 결정합니다.

session 값동작
비움(미전송)1회성 — 응답 직후 워커를 닫습니다. 매 호출이 새 워커(컨텍스트 없음).
임의 문자열(예: chat-42)keepalive — 그 워커를 살려두고, 같은 값으로 다시 호출하면 같은 워커가 컨텍스트를 유지한 채 이어서 답합니다.

예: turn1 session=chat-42 "비밀 단어는 바나나73, OK?" → OK · turn2 session=chat-42 "비밀 단어 뭐였지?" → 바나나73(기억함). session을 비우면 turn2는 기억하지 못합니다.

$h = @{ Authorization = 'Bearer <토큰>' }
# 1턴
Invoke-RestMethod -Method Post -Uri 'http://127.0.0.1:9610/run' -Headers $h -ContentType 'application/json' `
  -Body '{"agent":"claude","session":"chat-42","prompt":"내 이름은 Zendy. OK라고만 답해"}'
# 2턴 — 같은 session → 이름 기억
Invoke-RestMethod -Method Post -Uri 'http://127.0.0.1:9610/run' -Headers $h -ContentType 'application/json' `
  -Body '{"agent":"claude","session":"chat-42","prompt":"내 이름이 뭐였지?"}'

세션 종료: 같은 session을 더 안 쓰면 워커는 남아 있다가 앱 재시작/워커 탭 닫기로 정리됩니다(명시적 close API는 게이트웨이엔 없음 — 다회 대화는 오케스트레이션 MCP의 close_session 사용).

옵션 · 필드 매뉴얼

요청 본문 필드 (POST /run)

필드타입기본설명
agentstring—엔진: claude·codex·agy. (ai는 레거시 별칭)
aistring—레거시 별칭. 우선순위 agent > ai > "claude".
promptstring—프롬프트(우선). 없으면 messages 사용. 둘 다 없으면 400.
messagesarray—OpenAI식 [{role,content}]. content만 추출해 \n\n로 결합.
modelstring에이전트 기본선택. 에이전트에 전달.
sessionstring비움=1회성위 표 참조 — 값 지정 시 keepalive(멀티턴).

설정 화면 옵션 (설정 → AI 게이트웨이)

옵션기본설명
게이트웨이 사용신규 설치 onHTTP 서버 on/off. 변경 시 앱 재시작 후 적용.
포트9610기본 9610. 설정에서 변경할 수 있습니다.
토큰랜덤 48hexAuthorization: Bearer <토큰>. 복사 아이콘으로 복사.
워커 폴더<datadir>/gateway워커 실행 폴더(고정). 비우면 기본값. 바꾸면 새 폴더 1회 트러스트 승인.
워커 위험 권한 우회off(안전)off=위험 플래그 미사용(최초 1회 트러스트 승인 필요). on=무인 우회(빠름·비권장).
에이전트 실행을 탭으로 보기offexec 에이전트 출력을 탭으로 표시.

엔드포인트 · 응답 · 상태 코드

항목값
