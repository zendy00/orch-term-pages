오케스트레이션 게이트웨이 다운로드

Orch term은 여러 AI 코딩 에이전트(claude · codex · agy)를 한 곳에서 부려 조율하는 멀티에이전트 워크스테이션입니다. 이 문서는 오케스트레이션(지휘자) 기능 — 대화 중인 AI가 다른 AI를 워커로 띄워 다중 턴으로 협업시키는 기능 — 만 다룹니다.

외부 프로그램이 HTTP(OpenAI 호환)로 AI를 호출하는 AI 게이트웨이는 성격이 다른 기능입니다 → AI 게이트웨이 소개 페이지를 보세요.

1Orch term이란

터미널·코드 에디터·내장 브라우저를 한 창에서 쓰는 데스크톱 앱입니다. 핵심 차별점은 로컬 멀티에이전트 control plane — 외부(또는 내부)의 AI가 Orch term 안에서 다른 AI를 작업자(워커)로 띄우고, 격리된 작업 공간에서 일을 시키고, 결과를 받아 조율할 수 있습니다.

  • 지휘관(commander): 일을 지시하는 에이전트(보통 당신이 대화 중인 Claude).
  • 워커(worker): 지휘관이 띄운 codex·agy·claude 인스턴스. 각자 자기 탭/세션에서 실행.
  • 지휘자 대시보드: 살아있는 워커를 한눈에 보고 이동·종료하는 패널.

2오케스트레이션(지휘자)이란

외부 Claude Code(또는 내부 에이전트)가 MCP로 Orch term 안의 다른 AI를 인터랙티브 탭에 띄우고, 컨텍스트를 유지하며 다중 턴으로 대화·협업시키는 기능입니다.

지휘자 · MCP · 지속 세션

  • 프로토콜: MCP(orchterm-agent-orch), 포트 9611.
  • 도구: list_agents · open_session · send · close_session.
  • 워커를 인터랙티브 탭에 띄워 다중 턴 대화(컨텍스트 유지).
  • 적합: 한 작업자에게 여러 번 지시·되묻기, 워크트리 멀티에이전트 협업.
  • 보안: 127.0.0.1 바인딩 + Bearer 토큰, 기본 off(설정에서 켬).

3설정 방법

햄버거 메뉴 → 설정에서 켭니다. 활성화/포트 변경은 앱 재시작 후 적용됩니다(부팅 시 1회 배선).

항목설명
오케스트레이션(지휘자)토글로 켜면 MCP 서버(:9611)가 뜨고 토큰이 발급됩니다. 토큰은 ~/.claude.json 등에 자동 등록되어 외부 Claude Code가 바로 씁니다. 기본 off.
헤드리스 모드워커를 탭 없이(exec stdout) 돌릴지 여부. 끄면 워커가 보이는 탭으로 떠 진행을 눈으로 봅니다.

자동 등록 위치는 에이전트마다 다릅니다 — claude=~/.claude.json(인라인), codex=~/.codex/config.toml(토큰은 env), agy=~/.gemini 설정. 오케스트레이션 스킬도 세 에이전트의 skills 디렉터리에 함께 설치됩니다.

4사용법

워커 띄우기 (외부 Claude Code에서)

지휘자 MCP가 켜져 있으면, 대화 중인 Claude에게 자연어로 지시하면 됩니다. 예:

"워크트리 멀티에이전트로 A는 claude, B는 codex로 시켜줘.
 막히면 서로 결과를 넘겨서 조율해줘."

그러면 지휘관이 open_session으로 워커를 띄우고 send로 턴을 주고받습니다. 완료된 워크트리 작업의 머지는 항상 사용자 승인 후에만 이루어지도록 설계되어 있습니다.

최초 1회 폴더 승인 — 워커는 작업 폴더(보통 워크트리)에서 실행되는데, 각 에이전트가 그 폴더에서 처음 실행될 때 "이 폴더를 신뢰하시겠습니까?" 확인이 워커 탭에 한 번 뜹니다. 한 번 승인하면 그 폴더는 다시 묻지 않습니다(워크트리가 바뀌면 새로 1회). 무인으로 승인 없이 돌리려면 설정 → 에이전트 오케스트레이션 → "워커 위험 권한 우회"를 켜세요(빠르지만 비권장 — 기본은 안전 모드). (게이트웨이 워커 고정 폴더 안내는 게이트웨이 페이지 참고.)

지휘자 대시보드 열기 (보조 도구)

Ctrl+Shift+P로 명령 팔레트를 열고 “오케스트레이션 워커 지휘자”을 실행합니다. 살아있는 워커가 행으로 보이고, 각 행에서:

  • 상태: 작업중 / 완료 / 주의 / 대기.
  • 이동: 해당 워커 탭으로 점프(다른 Space에 있어도 따라감).
  • 닫기: 워커 세션·탭 종료. (워커 탭을 직접 닫아도 지휘자 목록에서 자동 정리됩니다.)

5스킬 소개

orchestrate-workers

메인 에이전트가 지휘자 MCP(orchterm-agent-orch)로 워커(codex·agy·claude)를 격리된 git 워크트리에 부려 조율하는 스킬입니다. 한 워커가 막혀 다른 산출물이 필요하면 메인이 다른 워커에게 위임하고 결과를 되돌려줍니다.

  • 트리거 예: “워크트리 멀티에이전트로”, “졸병 띄워서”, “워커에게 위임”, “A는 claude B는 codex로 시켜”.
  • 완료된 워크트리 작업은 준비(rebase·충돌해결·빌드/테스트)까지 자율, 머지는 사용자 승인 후.
  • 전제: orchterm-agent-orch MCP가 등록·실행 중이어야 합니다(설정에서 지휘자 활성화).

스킬은 지휘자을 켜고 앱을 재시작하면 claude·codex·agy 각 skills 디렉터리에 자동 설치됩니다. 세 에이전트 모두 동일한 SKILL.md를 네이티브로 읽습니다.
